## 1. 로그인 방식 결정

### 결론: 소셜 로그인 단독 채택

일반 로그인(이메일+비밀번호)과 소셜 로그인을 병행하는 방식 대신, **소셜 로그인만 사용**한다.

### 결정 근거

| 항목 | 내용 |
|---|---|
| **타겟 일치성** | 인디 아티스트, 유튜버, 아마추어 프로듀서 등 타겟 사용자는 Google/카카오 계정을 이미 보유하고 있음 |
| **공유 기능 연계** | YouTube, Instagram 등 소셜 공유 기능과 자연스럽게 연결됨 |
| **개발 복잡도 절감** | 비밀번호 암호화, 이메일 인증, 비밀번호 재설정 등 추가 구현 불필요 |
| **보안 위임** | 인증 보안 책임을 Google/카카오 등 OAuth 제공자에게 위임 가능 |
| **프로토타입 적합성** | 현재 단계는 워크플로우 검증이 목표이므로 로그인 시스템의 복잡도를 최소화하는 것이 적합 |

### 일반 로그인 병행이 필요한 경우 (향후 검토)

아래 상황에 해당될 경우 Phase 3~4에서 일반 로그인 추가를 고려한다.

- 소셜 계정이 없는 사용자(고령층, 기업 내부 사용자)를 포함해야 할 때
- 기업 B2B 서비스로 확장 계획이 생겼을 때

## 2. 소셜 로그인 제공자 선택

### 채택 제공자

| 제공자 | 우선순위 | 선택 이유 |
|---|---|---|
| **Google (OAuth 2.0)** | ⭐ 필수 | 타겟 사용자 대부분 보유, YouTube 공유와 자연 연결 |
| **카카오 (Kakao Login)** | ✅ 권장 | 국내 사용자 대상 서비스이므로 추가 권장 |

### 향후 확장 고려 제공자

| 제공자 | 조건 |
|---|---|
| **네이버 (Naver Login)** | 국내 사용자 확장 시 |
| **GitHub (OAuth)** | 개발자 커뮤니티 대상 확장 시 |
| **Apple Sign In** | iOS 앱 확장 시 (Apple 정책상 필수, Apple Developer 계정 $99/년 필요) |

## 3. 비용 구조

### 소셜 로그인 API 자체 비용: 무료

| 제공자 | 비용 | 비고 |
|---|---|---|
| Google OAuth 2.0 | 🟢 무료 | 호출 횟수 제한 없음 |
| 카카오 로그인 | 🟢 무료 | 호출 횟수 제한 없음 |
| 네이버 로그인 | 🟢 무료 | 호출 횟수 제한 없음 |
| GitHub OAuth | 🟢 무료 | 호출 횟수 제한 없음 |
| Apple Sign In | 🟢 무료 | Apple Developer 계정 별도 필요 |

> 소셜 로그인 API 호출 자체에는 구현자에게 비용이 발생하지 않는다.

### 주변 인프라 비용 (실제 비용 발생 지점)

| 항목 | 설명 | 비용 |
|---|---|---|
| **세션/토큰 저장소** | 로그인 후 사용자 세션 저장 (Redis, DB 등) | 소규모는 무료 티어로 충분 |
| **백엔드 서버** | OAuth 콜백 처리용 서버 필요 | Vercel, Supabase 등 무료 티어 활용 가능 |
| **HTTPS 인증서** | OAuth는 HTTPS 필수 | Let's Encrypt로 무료 발급 가능 |

## 4. 권장 기술 스택 (프로토타입 기준)

소셜 로그인부터 사용자 정보 저장, 배포까지 **$0 구성** 가능한 스택이다.

```
[인증 흐름]

Google / 카카오 OAuth (무료)
        ↓
NextAuth.js 또는 Supabase Auth    ← 소셜 로그인 처리 라이브러리 (무료)
        ↓
Supabase DB (무료 티어, 500MB)   ← 사용자 정보 저장
        ↓
Vercel 배포 (무료 티어)
```

### 스택 선택 이유

| 기술 | 이유 |
|---|---|
| **NextAuth.js** | Google, 카카오 등 다양한 OAuth 제공자를 플러그인 방식으로 간단히 연결 가능 |
| **Supabase Auth** | 백엔드 없이 소셜 로그인 + DB 저장을 한 번에 처리, 무료 티어 충분 |
| **Vercel** | Next.js 기반 프로젝트 최적 배포 환경, HTTPS 자동 적용 |

## 5. 유저 플로우 연동

유저플로우 문서(`유저플로우.md`)의 **s1 인증/온보딩** 서브그래프와 연동된다.

```
시작
  └─ 온보딩/로그인 화면
        ├─ Google 로그인 버튼 클릭 → Google OAuth → 콜백 → 세션 생성 → 홈 대시보드
        └─ 카카오 로그인 버튼 클릭 → 카카오 OAuth → 콜백 → 세션 생성 → 홈 대시보드
```

> 이메일/비밀번호 로그인 경로는 현재 스코프에서 제외한다.
